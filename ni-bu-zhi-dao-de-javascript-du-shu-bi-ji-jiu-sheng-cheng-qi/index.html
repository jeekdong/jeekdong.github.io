《你不知道的JavaScript》读书笔记（九）----生成器

  • 打破完整运行：在ES6中引入了一个新的函数类型，它并不符合运行到结束的特性

    • 输入与输出：当我们使用foo( 6,7 )只是创建了一个迭代器对象，并没有像普通函数一样实际运行

      • yield会导致生成器在执行过程中发出一个值，这有点类似于处于中间的return

  • 迭代消息传递：

function *foo(x) { //这是一个生成器
var y = x * (yield);
return y;
}

var it = foo( 6 );

// 启动 foo(..)
it.next();

var res = it.next( 7 ); //此处传入一个值 7 给 yield

res.value; // 42

yield作为一个表达式可以发出消息响应next(...)调用，next(...)也可以向暂停的yield表达式发送值。由此构成了一个双向消息传递系统

  • 多个迭代器：实际每次构建一个迭代器，实际就隐式构建了生成器的一个实例，而多个实例可以同时运行，他们甚至可以彼此交互

  • 生成器产生值

    • 生产者与迭代器：通过迭代器来产生一系列互有联系的值。for...of循环可以自动迭代标准迭代器，许多JavaScript内建的数据结构，比如array，也有默认的迭代器，一般的的对象没有迭代器，不过可以通过Object.keys(...)返回一个array
    • iterable：要想从一个iterable提取出一个迭代器,iterable必须支持Symbol.iterator函数，调用它，它会返回一个迭代器。如：var a = [1,3,5,7]; var it = a[Symbol.iterator]()
    • 生成器迭代器：因为生成器会在每个yield处暂停，函数 *something()的状态(作用域)会被保持，即意味着不需要闭包在调用之间保持变量状态
      • 停止迭代器：for...of循环有一个循环的"异常结束"，通常由break，return，或者未捕获异常引起(自动的)，也可以手工中止生成器的迭代器实例( it.return )

  • 异步迭代生成器：


function foo(x,y) {
	ajax(
		"http://some.url.1/?x=" + x + "&y=" + y,
		function(err,data) {
			if(err) {
				it.throw(err);
			} else { 
				it.next(data);        //从暂停处重新启动生成器
			}
		}
	);
}

function *main() {
	try {
		var text = yield foo(11,31);  //此处运行foo()，并等待ajax加载完数据
		console.log(text);
	} catch(err) {
		console.log(err);
	}
}

var it = main();

//启动生成器
it.next();

it.throw( err )可以把错误抛到生成器里，由生成器内部的try...catch(...)语句来处理，当生成器向外部抛出错误时，也会被外部的同步的try...catch(...)语句捕获并处理。当抛向生成器的错误没有被处理，应当由并且也会被 迭代器 的代码处理

  • 生成器+Promise

function foo(x,y) {
	return request("http://some.url.1/?x=" + x + "&y=" + y)  //它会返回一个Promise
}

function *main() {       //生成器完全与只有自己使用的代码一样，无需更改
	try {
		var text = yield foo(10,31);
		console.log( text );
	} catch (err) {
		console.log( err );
	}
}

//以下代码用来实现接受和连接yeiled出来的promise，使Promise决议出来后恢复生成器
var it = main();
var p = it.next().value;

//等待promise p决议
p.then(function(text){
		it.next( text );
	},
	function(err) {
		it.throw(err);
	}
);
  • 生成器中的Promise并发：

//原来代码如下,这里r2的请求要等到r1发送完才可以请求
function *foo() {
  var r1 = yield request("http://some.url.1");
  var r2 = yield request("http://some.url.1");
  
  var r3 = yield request("http://some.url.3/?v=" + r1 +"," + r2);
  ....
}

//改进后的
function *foo() {
  //这两个请求是并行的
  var p1 = request("http://some.url.1");
  var p2 = request("http://some.url.2");
  //等待两个Promise都决议
  var r1 = yield p1;
  var r2 = yield p2;
  
  var r3 = yield request("http://some.url.3/?v=" + r1 +"," + r2);
}

//也可以这样
function *foo() {
  //这两个请求是并行的
  var results = yield Promise.all([
  	request("http://some.url.1");
  	request("http://some.url.2");
  ]);
  //等待两个Promise都决议
  var [r1,r2] = results;
  
  var r3 = yield request("http://some.url.3/?v=" + r1 +"," + r2);
}

  • 生成器委托

    • 如果从一个run(...)调用中yield出来一个promise到另一个run(...)实例中，它会自动暂停*bar()，直到*foo()

    • 更好的做法使用yield委托(使用yield *___)


function *foo() {
	console.log( "`*foo()` starting" );
	yield 3;
	yield 4;
	console.log( "`*foo()` finished" );
}

function *bar() {
	yield 1;
	yield 2;
	yield *foo();	// `yield`-委托!
	yield 5;
}

var it = bar();

it.next().value;	// 1
it.next().value;	// 2
it.next().value;	// `*foo()` starting
				   // 3
it.next().value;	// 4
it.next().value;	// `*foo()` finished
				   // 5

    • yield *把迭代器实例控制委托给另一个*foo()迭代器，一旦it迭代器控制消耗了整个*foo()迭代器，it就会自动或转回控制*bar()

    • 实际上，yield可以委托到任意iterable，yield *[1,2,3]会消耗数组值[1,2,3]默认的迭代器

    • 消息的委托

      • 即使是被调用的*foo()，yield出来的值，也会被传递到it.next.value
      • 但是*foo的return的值会被作为*bar内部等待的yield *foo()表达式的结果值发出
      • 实际上，yield委托也可以委托到一个非生成器的iterable
      • 异常也会被委托！
      • 迭代委托（没看懂...(｡•́︿•̀｡)）

  • 生成器并发

    • 两种实现的手工方法

//第一种
// `request(..)` 是一个支持Promise的Ajax工具

var res = [];

function *reqData(url) {
	res.push(
		yield request( url )
	);
}
var it1 = reqData( "http://some.url.1" );
var it2 = reqData( "http://some.url.2" );

var p1 = it1.next().value;      //顺便提一下，这个地方，书上写错了，要带上属性`.value`
var p2 = it2.next().value;

p1                              //p1决议后首先恢复第一个实例，
.then( function(data){          
	it1.next( data );          
	return p2;                  //等数据存入数组第一位，返回p2
} )
.then( function(data){          //p2决议存入第二个数据
	it2.next( data );
} );

-----------------------------------------------------------------
  
//第二种
// `request(..)` 是一个支持Promise的Ajax工具

var res = [];

function *reqData(url) {
	var data = yield request( url );

	// 控制转移
	yield;

	res.push( data );
}

var it1 = reqData( "http://some.url.1" );
var it2 = reqData( "http://some.url.2" );

var p1 = it1.next().value;
var p2 = it2.next().value;

p1.then( function(data){
	it1.next( data );
} );

p2.then( function(data){
	it2.next( data );
} );                        //到此处两个迭代器都停到了那个第二个yield的地方，还没有存数据

Promise.all( [p1,p2] )      //等两个Ajax响应完毕，先存第一个实例到res[0]，第二个实例到res[1]
.then( function(){
	it1.next();
	it2.next();
} );

    • 两个方法最大的区别在于第一个方法，直到第一个实例完全结束才得到数据，而第二个方法两个示例都是各自响应一回来就取得了数据，在Promise.all( [...] )中处理恢复它们的顺序

    • 可以写一个runAll(...)的工具整合前面的手动方法(并没有看懂.(｡•́︿•̀｡))

  • 形式转换程序(thunk)

    • 狭义的表述：JavaScript中的thunk是指一个用于调用另外一个函数的函数，没有任何参数

    • 封装一个函数来实现自动创建thunk


function thunkify(fn) {
	return function() {                              //这里返回一个用于创建thunk的函数
		var args = [].slice.call( arguments );
		return function(cb) {
			args.push( cb );
			return fn.apply( null, args );
		};
	};
}

var whatIsThis = thunkify( foo );         //这里`whatIsThis`引用的是一个thunk的工厂

var fooThunk = whatIsThis( 3, 4 );

// later

fooThunk( function(sum) {
	console.log( sum );		// 7
} );
  • thunkory和promisory本质上都提出了一个请求，分别有thunk fooThunk和promise fooPromise表示对这个请求的未来的答复
