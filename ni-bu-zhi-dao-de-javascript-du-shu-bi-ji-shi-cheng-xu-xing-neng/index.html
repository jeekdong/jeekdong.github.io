《你不知道的JavaScript》读书笔记（十）----程序性能

  • Web Worker

    • 程序中每一个独立的多线程部分被称为(Web) Worker ，从JavaScript的主程序(或另一个Worker)，可以实例化一个Worker：var w1 = new Worker("http://some.url.1/mycoolworker.js")，这个URL应当指向一个JavaScript文件的位置，而不是一个HTML文件的位置，这个文件将在独立的线程中工作

    • Worker之间以及它们和主程序之间只是通过一个基本的事件消息机制相互联系

    
  w1.addEventListener("message",function(evt) {
    //evt.data
  });

    //发送"message"事件给这个Worker
    
w1.postMessage("somenthing coll to say")
    

    //中止Worker
    
w1.terminate();

    • Worker环境：在Worker内部是无法访问主程序的任何资源(全局变量，DOM)，但是可以执行一些网络操作(Ajax，WebSockets)

    • 数据传递：早期只能把所有数据序列化到一个字符串值，浪费性能，现在可以使用结构化克隆算法，Transferable对象

    • 共享Worker：

      • 通过以下方式创建var w1 = new SharedWorker("http://some.url.1/mycoolworker.js")
      • 通过端口来标志与该worker连接的不同的程序
      • 调用时的注意事项
        1. 必须使用Worker的port对象用于通信w1.port.postMessage(...)
        2. 端口连接必须要初始化：w1.port.start()
        3. 共享Worker内部要处理一个事件："connect",它为特定的连接提供了端口对象
      • 专用Worker，只要实例化它的程序的连接终止，它就会终止，而共享Worker只要还有端口活跃，就不会终止

  • SIMD(单指令多数据)

    • SIMD是一种数据并行的方式，它并行处理数据的多个位，不提供线程上的并行，利用低级指令集并行的底层运算
