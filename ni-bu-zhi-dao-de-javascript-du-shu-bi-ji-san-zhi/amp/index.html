《你不知道的JavaScript》读书笔记（三）----值

by jeekdong

2017-06-22

  • 使用delete运算符可以将单元从数组中删除，但是不改变length属性的值

  • 数组也是对象的子类型，所以也可以包含字符串键值和属性(但不计算在数组长度内)

  • 类数组---->数组：使用slice()和Array.from()

  • JavaScript字符串不可变，数组可变。许多数组函数用来处理字符串很方便，可以通过"借用"数组的非变更方法来处理字符串，

    
a.join;			// undefined
a.map;			// undefined

    var c = Array.prototype.join.call( a, "-" );
    
var d = Array.prototype.map.call( a, function(v){
    
return v.toUpperCase() + ".";
    
} ).join( "" );
    

    c;				// "f-o-o"
    
d;				// "F.O.O."

  • split()：字符串--->数组。join()：数组---->字符串。

  • JavaScript无真正意义上的整数

  • 42.toFixed(3) //无效 因为 . 会被视为小数点而不是属性运算符；应当使用42..toFixed(3)或者(42).toFixed(3)

  • 0x(十六进制) , 0/0o(ES6的新表达方式) (八进制) , 0b(二进制)

  • 通过设置“机器精度”(存储在Number.EPSILON)可以用来比较 0.1+0.2 与0.3是否相等

  • null 与 undefined 的区别：

    • null指空值 / 目前没有值 null不是标识符，不可以当作变量来使用和赋值
    • undefiend 指从未赋值 undefined是标识符，可以赋值

  • void运算符可以使表达式不返回值(设为undifined)

  • 如果isNaN函数的参数不是Number类型,isNaN()会首先尝试将这个参数转换为数值，然后才会对转换后的结果是否是NaN进行判断。可以把isNaN看做：


  isNaN = function(value) {
      Number.isNaN(Number(value));
  }

在JavaScript中，isNaN(x)==true等价于x - 0 = NaN,相当于它是一个不可运算数，可以利用这个特殊行为来检测函数的参数是可运算的

  • 在JavaScript中 0 === -0 //true，所以可以使用Object.is()来判断(也可以判断两个NaN是否相等)，Object.is(0,-0) //false

  • 简单值(6种),总是通过值复制的方式来赋值/传递，复合值------对象，则总是通过引用复制的方式来赋值/传递，

  • 在JavaScript中，如果参数不声明为引用的话，参数值总是通过值复制的方式传递，即便对复杂的对象值也是如此（意味着我们不能通过引用x来改变a的指向，只能更改a和x共同指向的值）。也可以说是，在JavaScript中的引用和其他寓言中的引用/指针不同，他们不能指向别的变量/引用，只能指向值
