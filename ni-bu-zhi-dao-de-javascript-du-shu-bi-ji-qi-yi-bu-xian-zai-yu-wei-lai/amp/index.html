《你不知道的JavaScript》读书笔记（七）----异步：现在与未来

by jeekdong

2017-07-17

  • 异步控制台：在某些浏览器的console.log(...)并不会把传入的内容立即输出（因为，I/O是非常低速的阻塞）

  • 事件循环：在ES6以前JS引擎并没有内建直接的异步概念，而是通过事件循环


  // `事件循环` 是一个用作队列的数组
  var eventLoop = [ ];
  var event;

  // 永远执行
  while (true) {
  	// 一次"tick"
  	if (eventLoop.length > 0) {
  		// 得到下一个队列的事件
  		event = eventLoop.shift();
  		// 现在，执行下一个事件
  		try {
  			event();
  		}
  		catch (err) {
  			reportError(err);
  		}
  	}
  }

有一个while循环是现代持续运行的循环，循环的每一轮成为一个tick，对每个tick而言，如果在队列中有事件，那么就会从队列中得到一个事件并执行。这些事件就是回调函数。

  • 并行线程：运行中会出现中断和交错运行

  • 完整运行：由于JavaScript是单线程的，一旦foo()开始运行，他的所有代码都会在bar()的任意代码运行之前完成。

  • 并发：可以把"并发"看做“进程”级的并行，与运算级的并行相对。单线程事件循环是并发的一种形式

    • 交互：由于异步的不确定性，可以通过协调交互顺序来处理这样的竞争条件

    var res = [];
    function response(data) {
    	if (data.url == "http://some.url.1") {
    		res[0] = data;
    	}
    	else if (data.url == "http://some.url.2") {
    		res[1] = data;
    	}
    }
    // ajax(..) is some arbitrary Ajax function given by a library
    ajax( "http://some.url.1", response );
    ajax( "http://some.url.2", response );

不管哪一个Ajax响应先返回，我们都要通过查看data.url，判断应该吧相应数据放在res数组的什么位置

  • 协作：如果有一个长期运行的“进程”，可以将其分割成多个步骤或多批任务。让它找机会将自己的运算插入到事件循环队列中交替运行

  • 任务：ES6中有一个建立在事件循环队列之上的概念，“任务队列”。

    • 任务队列它是挂在事件循环队列的每个tick之后的一个队列。在一个事件的tick中，如果发生了一个异步事件，他不会导致一个新事件被添加到事件循环的队列中，而是在这个tick的任务队列末尾添加一个任务

    console.log( "A" );
    setTimeout( function(){
    	console.log( "B" );
    }, 0 );
    // theoretical "Job API"
    schedule( function(){
    	console.log( "C" );
    	schedule( function(){
    		console.log( "D" );
    	} );
    } );

上面的例子最后输出的结果是A C D B。

  • Promise的异步特性是基于任务的
