《你不知道的JavaScript》读书笔记（六）----语法

  • 语句的结果值：

    • 赋值表达式的结果是赋值的值
    • 代码块的结果值是其最后一个语句/表达式的结果
    • 但是语法并不允许我们获得语句的结果值并将其赋值给另一个变量，不过eval()可以获得结果值，ES7也有一个提案“do表达式”，会返回其中最后一个语句的结果值
    • javascript var a = b = 42; //这个语句并不会对b进行声明

  • 上下文规则：

    • 代码块:第一行的{}被当做空对象，第二行的{}被当做代码块

    [] + {}     // "[object Object]"
    {} + []     // 0
  • 对象解构：从ES6开始，{...}可以用于解构赋值

    function getData() {
    	// ..
    	return {
    		a: 42,
    		b: "foo"
    	};
    }

    var { a, b } = getData();

    console.log( a, b ); // 42 "foo"

{a,b}实际就是{a:a,b:b}的简化版本

也可以用作函数命名参数的对象解构

  • JavaScript中没有else if，我们看到的else if实际是：

   if (a) {
     // ..
    }else {
     if (b) {
      // ..
     }else {
      // ..
     }
   }

  • 运算优先级&关联性

  • 自动分号：javaScript会自动在代码行补上缺失的分号，例如do...while{...}循环后面必须带;还有break，continue，return，yield(ES6)；

  • 错误：

    • 提前使用变量：ES6有个概念：TDZ(暂时性死区)，会抛出ReferenceError！但是对未声明变量执行typeof不会产生错误，但在TDZ中却会报错

  • 函数参数：另一个TDZ违规的例子就是ES6中的参数默认值，并且传入的参数被省略或者值为undefined，则取该参数的默认值，在向函数传递参数时，arguments数组中的对应单元会和命名参数建立关联

  • try...finally

    • finally中的代码总是会在try之后执行，如果有catch则在catch之后执行，无论如何都会执行。
    • 如果finally中抛出了异常，函数就会再次终止，即使有return值也会被抛弃
    • finally中的return会覆盖try和catch中的return返回值

  • switch

    • 参数与case表达式的匹配算法与 === 相同，
