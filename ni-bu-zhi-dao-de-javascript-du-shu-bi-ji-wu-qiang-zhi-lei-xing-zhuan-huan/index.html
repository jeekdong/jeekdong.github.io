《你不知道的JavaScript》读书笔记（五）----强制类型转换

  • ToString操作

    • null--->"null",undefined--->"undefined",true--->"true"
    • 普通对象----->返回内部[[class]]值("[object object]")
    • 数组---->"1,2,3"
    • JSON.stringify(...)：字符串，数字，布尔值和null规则与ToString操作相同

  • ToNumber操作

    Argument Type Result
    Undefined NaN
    Null +0
    Boolean The result is 1 if the argument is true.The result is +0 if the argument is false.
    Number The result equals the input argument (no conversion).
    String In effect evaluates Number( string)“abc” -> NaN“123” -> 123, 空字符串，" ", "\n"会被转换为0
    Object Apply the following steps:1. Let primValue be ToPrimitive( input argument, hint Number).2. Return ToNumber( primValue). 有valueOf()就执行（数组的valueOf()会返回本身），否则执行toString(),否则报错

  • ToBoolean操作

    • 假值表：undefined,null,false,+0,-0,NaN,"" 假值列表以外的值都是真值

  • 一元运算符"+"可以将日期对象转换为数字

  • ~按位非，返回2的补码(大致相当于-(42+1)) P59

  • parseInt()解析字符串中的数字与字符串强制类型转换有区别，解析允许字符串有非数字字符，从左至右。而转换不允许，否则会失败返回NaN

  • 显示转换为布尔值：Boolean(...)，!!a

  • 字符串与数字之间隐式强制转换：+运算符可用于字符串拼接和数字加法；var a = [1,2]; var b = [3,4]; a+b;//"1,23,4" 它会对数组执行ToPrimitive操作，a+""这样的隐式操作与String有一点区别，前者valueOf---->ToString，后者直接ToString

  • 布尔值到数字的隐式强制转换： +运算符 var a = true +true //2

  • 隐式强制类型转换为布尔值：

    • if(...)语句中的判断表达式
    • for(...)语句中的判断表达式
    • while，do...while 中的判断表达式
    • ? : 中的条件判断表达式
    • 逻辑运算符||和&&左边的操作数(作为条件判断表达式)

  • ||和&&，可以叫为“选择器运算符”，因为在JavaScript中它们返回的并不是布尔值，而是两个操作数中的一个(且仅一个)

    • ||会首先对第一个条件判断，非布尔值执行ToBoolean，true返回第一个操作数，false返回第二个操作数
    • &&会首先对第一个条件判断，非布尔值执行ToBoolean，true返回第二个操作数，true返回第一个操作数

  • ES6允许从符号到字符串的显式强制类型转换，然而隐式强制类型转换会产生错误

  • 相等 (｡•́︿•̀｡)：

    • String == Boolean，需要两个操作数同时转为Number。
    • JavaScript中对象的比较是引用的比较，而不是值的比较
    • JS中奇怪的地方：
      1. 对象在等于操作符中会执行valueOf()方法，可以在此返回其他数字，可产生与预想不一样的结果
      2. if(a == 2 && a == 3){ ... }看起来这个式子不可能执行，因为a不会同时等于2和3.但"同时"一词并不准确，因为a == 2在a == 3之前执行。如果让a.valueOf()每次调用产生副作用就可以实现。

  • 不等比较：

    • a<=b 会被处理为 a>b ,然后将结果反转
