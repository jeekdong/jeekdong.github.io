ubuntu 关于Hash校验和不符的解决

问题

最近需要在ubuntu14.04中安装shadowsocks，但是每次使用sudo apt-get update来更新源软件时都会显示Hash校验和不符，无法下载。

再使用sudo apt-get install shadowsocks-qt5来安装便会提示缺少依赖，因为Update并没有更新我们新加源的软件。

原因

这就很奇怪，一开始我还以为自己的网络原因，或是服务器抽风，隔了一天继续尝试，依然无法更新，上网谷歌了一翻，才发现原来是运营商缓存的问题，出现这个错误的原因是 Ubuntu 下载的索引文件不是来自指定的软件源，而是网络服务提供商的缓存。

随着中国互联网网站和网民数量的剧增，互联网用户经常会遇到网站服务器因超负载链接而出现反应时间很慢的情况。对于网站运营商来说，增加昂贵的带宽并不是唯一的解决方法，有一种费用相对较为低廉，而效果又非常理想的服务方式正在广为人们所接受，那就是Web高速缓存技术。
Web高速缓存基于以下事实: 特定的WWW对象经常被多个网络用户再次请求时，缓存设备会监视Web请求，检索它们，当第一次传送对象时把它存储到缓存设备中, 在以后的请求中发送缓存设备中(而不是目标站点)的对象。

「校验和不符」（ Hash Sum mismatch） 在 2012 年已经被确认是 Ubuntu 的一个 BUG ， 但是几年过去，还是没改好。

解决

具体解决办法我是采用HTTP代理的方式。HTTP代理自行谷歌，有很多免费的，记得使用前ping一下，测试连通性。但是这里有个坑，网上流传较广的临时代理的手段，export http_proxy=http://yourproxyaddress:proxyport是行不通的。

真正的解决方法如下：

  • 执行sudo gedit /etc/apt/apt.conf在apt.conf文件中加入下面这行Acquire::http::Proxy "http://yourproxyaddress:proxyport";保存并关闭。
  • 再执行sudo apt-get update即可正常更新
  • 更新完毕后删除刚才添加的那一行
