《你不知道的JavaScript》读书笔记（二）----对象原型&类型(type)

1.对象原型

  • 属性访问时执行[[Get]]操作：

    • 对象默认内置的[[Get]]操作首先在对象中查找是否有名称相同的的属性，如果找到就返回
    • 如果没有找到名称相同的属性，会去遍历可能存在的[[Prototype]]链

  • 属性访问时的[[Put]]操作（存在这个属性）：

    • 属性是否是访问描述符？如果是并且存在setter就调用setter
    • 属性的数据描述符中writable是否是false？如果是，非严格模式下会静默失败
    • 如果都不是，就将该值设置为属性的值

  • 在原型链上层时myObject.foo = "bar"会出现的三种情况

    1. 如果在[[prototype]]链上层存在名为foo的普通数据访问属性并且没有被标记为只读(writable：flase)，那就会直接在myObject上添加一个名为foo的新属性，他是屏蔽属性

    2. 如果在[[prototype]]链上层存在foo，但是它被标记为只读(writable: true)，那么无法修改已有属性或者在myObject上创建屏蔽属性。一般这条语句会被忽略。

    3. 如果在[[prototype]]链上层存在foo并且它是一个setter，那就一定会调用这个setter。foo不会被添加到myObject，也不会重新定义这个setter。

      如果希望在第二和第三种情况下也屏蔽foo，那就不能使用 = 赋值符，而是使用Object.defineProperty(...)来向myObject中添加foo

  • 注意隐式屏蔽（myObject.a++），所以修改委托属性是一定要小心。

  • new Foo()只是间接关联了对象，这个关联只是一个意外的副作用。

  • （原型）继承最好采用的方法是Object.create()或者用Object.setPrototypeOf()

  • 检查"类"的关系：

    • a instanceof Foo左边是一个普通的对象，右边是一个函数。它回答的问题是：在a的整条[[Prototype]]链中是否有指向Foo.prototype的对象
    • Foo.prototype.isPrototypeOf( a )左右两边都是对象，它回答的问题是：在a的整条[[Prototype]]链中是否出现过Foo.prototype
    • ES5中可以直接获取一个对象的[[Prototype]]链 Object.getPrototypeOf( a )

  • Object.create()是通过Prototype来连接的

2.类型(type)

  • JavaScript有7种内置类型：ES6新增了symbol

  • null:：typeof 对null的处理有点问题：typeof null === "object" //true ；null是基本类型中唯一一个“假值“

  • function是对象子类型，函数对象的length属性是其声明的参数的个数

    
function a(b,c){...};
a.length;    //2

  • JavaScript中的变量是没有类型的，只有值才有，typeof 运算符总会返回字符串，且typeof 未声明的变量也是undefined（所以要注意区分），这是因为typeof有一个特殊的安全防范机制

  • typeof 的安全防范机制可用来检测变量是否定义过，也可以用window.variable来检测，还有一种依赖注入设计模式。

    
// 这样会抛出错误
if (DEBUG) {
	console.log( "Debugging is starting" );
}

    // 这样是安全的
    
if (typeof DEBUG !== "undefined") {
    
console.log( "Debugging is starting" );
    
}
